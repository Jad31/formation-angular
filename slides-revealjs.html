SIGNALS

Réactivité nouvelle génération

Pourquoi Signals ?

Problème Zone.js :

  • Vérifie TOUT à chaque événement
  • Performance dégradée
  • Comportement imprévisible

Solution Signals

  • Réactivité précise
  • Mise à jour ciblée
  • Code explicite
  • Meilleure performance

signal() - Créer un signal

import { signal } from '@angular/core';

// Signal simple
count = signal(0);

// Signal type
name = signal<string>('Angular');

// Signal avec objet
user = signal({ name: 'John', age: 30 });

Lire un signal

count = signal(5);

// Dans la classe
console.log(this.count()); // 5
<!-- Dans le template -->
<p>{{ count() }}</p>

Toujours avec ()

Modifier - set()

count = signal(0);

// Remplacer la valeur
this.count.set(10);

// Reinitialiser
this.count.set(0);

Modifier - update()

count = signal(0);

// Incrementer
this.count.update(n => n + 1);

// Decrementer
this.count.update(n => n - 1);

// Doubler
this.count.update(n => n * 2);

computed() - Valeurs dérivées

import { signal, computed } from '@angular/core';

firstName = signal('John');
lastName = signal('Doe');

// Valeur calculée automatiquement
fullName = computed(() =>
  `${this.firstName()} ${this.lastName()}`
);

computed() est en lecture seule

count = signal(5);
double = computed(() => this.count() * 2);

// Lecture OK
console.log(this.double()); // 10

// ERREUR - pas de set() ! - 
this.double.set(20); // TypeError

effect() - Side effects

import { signal, effect } from '@angular/core';

count = signal(0);

constructor() {
  effect(() => {
    console.log('Count:', this.count());
  });
}

S'exécute à chaque changement de count

effect() - Cas d'utilisation

  • Synchronisation localStorage
  • Logging / analytics
  • Appels API déclenchés
  • Mise a jour du DOM manuel

Exemple Counter complet

@Component({
  template: `
    <p>Count: {{ count() }}</p>
    <p>Double: {{ double() }}</p>
    <button (click)="increment()">+</button>
  `
})
export class CounterComponent {
  count = signal(0);
  double = computed(() => this.count() * 2);

  increment() { this.count.update(n => n + 1); }
}

Signals + Objets

user = signal({ name: 'John', age: 30 });

// Mise a jour immutable
updateAge(newAge: number) {
  this.user.update(u => ({
    ...u,
    age: newAge
  }));
}

Toujours créer un nouvel objet

Signals + Arrays

items = signal<string[]>([]);

addItem(item: string) {
  this.items.update(arr => [...arr, item]);
}

removeItem(index: number) {
  this.items.update(arr =>
    arr.filter((_, i) => i !== index)
  );
}

  this.favorites.update(favs => {
    if (favs.includes(id)) {

    return [...favs, id];

}

Notification Service

@Injectable({ providedIn: 'root' })
export class NotificationService {
  private messages = signal<string[]>([]);
  readonly all = this.messages.asReadonly();

  show(message: string) {
    this.messages.update(m => [...m, message]);
    setTimeout(() => this.remove(0), 3000);
  }

  remove(index: number) {
    this.messages.update(m =>
      m.filter((_, i) => i !== index));

@Pipe({
  name: 'truncate',
  standalone: true
})
export class TruncatePipe implements PipeTransform {
  transform(value: string | null, limit = 50, ellipsis = '...'): string {
    if (!value) return '';
    if (value.length <= limit) return value;
    return value.substring(0, limit) + ellipsis;
  }
}

export class PokemonFilter {
  private fb = inject(FormBuilder);
  private pokemonService = inject(PokemonService);

  types = ['fire', 'water', 'grass', 'electric', 'psychic'];

  filterForm = this.fb.group({
    name: ['', Validators.minLength(2)],
    type: [''],
    minStats: [0, [Validators.min(0), Validators.max(255)]]
  });

  get name() { return this.filterForm.controls.name; }

  onFilter() {
    if (this.filterForm.valid) {
      const { name, type, minStats } = this.filterForm.value;
      this.pokemonService.applyFilters({ name, type, minStats });
    }
  }

  clearFilters() {
    this.filterForm.reset({ name: '', type: '', minStats: 0 });
    this.pokemonService.clearFilters();
  }
}
